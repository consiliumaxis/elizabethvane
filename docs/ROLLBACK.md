# Откат проекта к предыдущей версии

Автосинхронизация форка `consiliumaxis/elizabethvane` с upstream устроена так,
что **перед каждым обновлением** веток `main` и `test` (когда в upstream реально
появились новые коммиты) автоматически создаётся **тег-снимок** предыдущего
рабочего состояния. Это ваши точки восстановления.

## Как выглядят снимки

- `snapshot/main/ГГГГ-ММ-ДД-ЧЧММСС` — состояние ветки `main` перед синком (время в UTC)
- `snapshot/test/ГГГГ-ММ-ДД-ЧЧММСС` — состояние ветки `test` перед синком (время в UTC)

Хранятся последние 50 снимков на каждую ветку, старые удаляются автоматически.

## Посмотреть список точек восстановления

```bash
git fetch origin --tags --prune
git tag -l 'snapshot/main/*' | sort -r   # последние сверху
git tag -l 'snapshot/test/*' | sort -r
```

Также все снимки видны на GitHub: вкладка **Tags** репозитория форка.

## Откатить ветку на выбранный снимок

Пример — вернуть `main` на снимок от 20 июля:

```bash
git push origin 'refs/tags/snapshot/main/2026-07-20-183000:refs/heads/main' --force-with-lease
```

То же для `test` — подставьте нужный тег `snapshot/test/...`.

> Внимание: следующий автосинк снова подтянет актуальную версию upstream.
> Если проблема в самом upstream (там запушена сломанная версия), откат
> временный — сначала нужно дождаться исправления в upstream, а до тех пор
> можно временно отключить workflow (Actions → Sync Upstream → Disable workflow).

## Ручной бэкап в любой момент

```bash
git tag backup/$(date -u +%Y-%m-%d-%H%M%S)-ручной origin/main
git push origin --tags
```

## Важное правило

Не коммитьте напрямую в `main` и `test` — они жёсткие зеркала upstream, ваши
прямые коммиты будут стёрты при следующем синке. Вся своя разработка — только
в ветках `feature/*`.
