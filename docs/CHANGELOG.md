### Заметка по запуску тестов
Тесты в `backend/tests/` используют смешанные стили импорта:
- часть — `from <module>` (например `from pocket_api import ...`), требует `backend/` в `PYTHONPATH`;
- `test_access_policy.py` — `from backend.access_policy import ...`, требует корень репозитория в `PYTHONPATH`.

Чтобы все тесты собирались и проходили локально, оба пути нужно добавить в `PYTHONPATH`:
```bash
cd backend
PYTHONPATH="$(pwd)/..:$(pwd)" python -m pytest -q
```
На сервере это, вероятно, обеспечивается окружением деплоя. Отдельной задачей стоит унифицировать стиль импорта или добавить `conftest.py`, чтобы работал простой `cd backend && python -m pytest -q`.
